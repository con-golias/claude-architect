## Essential Books

| Book | Author | Level | Why Read It |
|------|--------|-------|-------------|
| **"The Go Programming Language"** | Donovan & Kernighan | Beginner-Intermediate | The definitive Go book — co-authored by Brian Kernighan (K&R C fame) |
| **"100 Go Mistakes and How to Avoid Them"** | Teiva Harsanyi | Intermediate | Practical error catalog, excellent for Go developers at any level |
| **"Concurrency in Go"** | Katherine Cox-Buday | Intermediate-Advanced | Deep dive into goroutines, channels, and concurrency patterns |
| **"Go in Action"** | Kennedy, Ketelsen, Martin | Beginner-Intermediate | Practical introduction, good for experienced developers learning Go |
| **"Let's Go" / "Let's Go Further"** | Alex Edwards | Intermediate | Building production web applications with Go |
| **"Learning Go"** (2nd ed.) | Jon Bodner | Beginner-Intermediate | Modern Go with generics, updated best practices |
| **"Network Programming with Go"** | Adam Woodbeck | Intermediate | TCP/UDP, HTTP, WebSocket in Go |
| **"Powerful Command-Line Apps in Go"** | Ricardo Gerardi | Intermediate | Building CLI tools |

## Learning Paths

### Beginner → Intermediate (1-2 months)

```
1. Go Tour (go.dev/tour) — interactive, free
2. Go by Example (gobyexample.com) — learn by doing
3. "The Go Programming Language" (book) — comprehensive
4. Build: CLI tool, REST API, concurrent file processor
```

### Intermediate → Advanced (2-4 months)

```
1. "100 Go Mistakes" (book) — avoid common pitfalls
2. "Concurrency in Go" (book) — master goroutines/channels
3. Study Go stdlib source code (net/http, encoding/json)
4. Profiling with pprof
5. Build: microservice with gRPC, worker pool, load balancer
```

### Expert (Ongoing)

```
1. Read Go compiler source code
2. GC internals, scheduler internals
3. Contribute to Go open source projects
4. Study runtime package
5. Write Go proposals
```
