## Essential Books

### Core Python

| Book | Author | Level | Why Read It |
|------|--------|-------|-------------|
| **"Fluent Python"** (2nd ed.) | Luciano Ramalho | Advanced | Deep dive into Python data model, metaclasses, concurrency, type hints — the definitive advanced Python book |
| **"Effective Python"** (2nd ed.) | Brett Slatkin | Intermediate-Advanced | 90 specific ways to write better Python, Pythonic idioms |
| **"Python Cookbook"** (3rd ed.) | David Beazley, Brian K. Jones | Intermediate | Practical recipes for common problems |
| **"Automate the Boring Stuff"** | Al Sweigart | Beginner | Free online, practical automation projects |
| **"Python Crash Course"** (3rd ed.) | Eric Matthes | Beginner | Best first Python book, project-based |
| **"Learning Python"** (5th ed.) | Mark Lutz | Beginner-Intermediate | Most comprehensive introduction (~1,600 pages) |

## Learning Paths

### Beginner → Intermediate (2-4 months)

```
1. Python Basics
   └── Python Crash Course (book) or Automate the Boring Stuff
   └── Official Python Tutorial (docs.python.org/3/tutorial/)

2. Intermediate Python
   └── Real Python tutorials
   └── List comprehensions, generators, decorators, context managers

3. Practical Projects
   └── Web scraping (BeautifulSoup, requests)
   └── CLI tool (Click or Typer)
   └── Simple API (FastAPI)
```

### Intermediate → Advanced (4-8 months)

```
1. Deep Python
   └── "Fluent Python" by Luciano Ramalho
   └── Data model, descriptors, metaclasses

2. Type System Mastery
   └── Type hints, mypy strict mode, Protocols
   └── "Robust Python" by Patrick Viafore

3. Async & Concurrency
   └── asyncio, structured concurrency (TaskGroup)
   └── multiprocessing, threading

4. Testing Mastery
   └── pytest, hypothesis, mocking
   └── "Python Testing with pytest" by Brian Okken
```

### Expert Level (Ongoing)

```
1. CPython Internals
   └── "CPython Internals" by Anthony Shaw
   └── Reading CPython source code

2. Performance
   └── Profiling, Cython, Numba
   └── "High Performance Python"

3. Architecture
   └── "Architecture Patterns with Python"
   └── DDD, event-driven, hexagonal

4. Contribution
   └── Open source contribution
   └── PEP process, CPython development
```
