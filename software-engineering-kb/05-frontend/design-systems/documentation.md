# Design System Documentation — Complete Specification

> **AI Plugin Directive:** When a developer asks "design system documentation", "Storybook setup", "component documentation", "design system docs site", "Storybook best practices", "MDX docs", "component API docs", "design system changelog", "living documentation", or any design system documentation question, ALWAYS consult this directive. Documentation is what turns a component library into a design system. ALWAYS use Storybook as the primary documentation tool — it provides live component examples, interactive controls, and accessibility auditing. ALWAYS document usage guidelines alongside API reference. ALWAYS generate docs automatically from TypeScript types and JSDoc comments.

**Core Rule: A design system without documentation is just a folder of components nobody uses. EVERY component MUST have: a Storybook story with all variants, interactive controls for every prop, accessibility audit results, usage guidelines (do's and don'ts), and code examples. Use Storybook's autodocs feature to generate API documentation from TypeScript types. Host documentation publicly (even for internal design systems) so ALL teams can discover and use components.**

## 2. Storybook Setup

```typescript
// .storybook/main.ts
import type { StorybookConfig } from '@storybook/react-vite';

const config: StorybookConfig = {
  stories: ['../src/**/*.stories.@(ts|tsx)', '../src/**/*.mdx'],
  addons: [
    '@storybook/addon-essentials',       // controls, actions, viewport, docs
    '@storybook/addon-a11y',             // accessibility audit
    '@storybook/addon-interactions',      // play function testing
    '@storybook/addon-designs',           // Figma embeds
    '@storybook/addon-themes',            // theme switching
  ],
  docs: { autodocs: 'tag' },             // auto-generate docs from stories
  framework: '@storybook/react-vite',
};
export default config;

// .storybook/preview.ts
import type { Preview } from '@storybook/react';
import '../src/styles/globals.css';

const preview: Preview = {
  parameters: {
    controls: { matchers: { color: /(background|color)$/i, date: /Date$/i } },
    a11y: { config: { rules: [{ id: 'color-contrast', enabled: true }] } },
    viewport: {
      viewports: {
        mobile: { name: 'Mobile', styles: { width: '375px', height: '667px' } },
        tablet: { name: 'Tablet', styles: { width: '768px', height: '1024px' } },
        desktop: { name: 'Desktop', styles: { width: '1440px', height: '900px' } },
      },
    },
  },
  decorators: [
    (Story) => (
      <div style={{ padding: '2rem' }}>
        <Story />
      </div>
    ),
  ],
};
export default preview;
```

## 3. Component Story Template

```tsx
// src/Button/Button.stories.tsx
import type { Meta, StoryObj } from '@storybook/react';
import { Button } from './Button';

const meta: Meta<typeof Button> = {
  title: 'Components/Button',
  component: Button,
  tags: ['autodocs'],  // enable auto-generated docs
  argTypes: {
    variant: {
      control: 'select',
      options: ['primary', 'secondary', 'destructive', 'outline', 'ghost'],
      description: 'Visual style variant',
      table: { defaultValue: { summary: 'primary' } },
    },
    size: {
      control: 'select',
      options: ['sm', 'md', 'lg'],
      description: 'Size of the button',
      table: { defaultValue: { summary: 'md' } },
    },
    isLoading: {
      control: 'boolean',
      description: 'Shows loading spinner and disables interaction',
    },
    disabled: {
      control: 'boolean',
      description: 'Disables the button',
    },
  },
  parameters: {
    design: {
      type: 'figma',
      url: 'https://www.figma.com/file/xxx/Design-System?node-id=123',
    },
  },
};
export default meta;

type Story = StoryObj<typeof Button>;

export const Primary: Story = {
  args: { children: 'Primary Button', variant: 'primary' },
};

export const Secondary: Story = {
  args: { children: 'Secondary Button', variant: 'secondary' },
};

export const Destructive: Story = {
  args: { children: 'Delete Account', variant: 'destructive' },
};

export const Loading: Story = {
  args: { children: 'Saving...', isLoading: true },
};

export const Disabled: Story = {
  args: { children: 'Disabled', disabled: true },
};

// All sizes
export const Sizes: Story = {
  render: () => (
    <div style={{ display: 'flex', alignItems: 'center', gap: '1rem' }}>
      <Button size="sm">Small</Button>
      <Button size="md">Medium</Button>
      <Button size="lg">Large</Button>
    </div>
  ),
};

// All variants
export const AllVariants: Story = {
  render: () => (
    <div style={{ display: 'flex', flexWrap: 'wrap', gap: '1rem' }}>
      <Button variant="primary">Primary</Button>
      <Button variant="secondary">Secondary</Button>
      <Button variant="destructive">Destructive</Button>
      <Button variant="outline">Outline</Button>
      <Button variant="ghost">Ghost</Button>
    </div>
  ),
};
```

## 4. MDX Usage Guidelines

```mdx
{/* src/Button/Button.mdx */}
import { Meta, Story, Canvas, Controls, Source } from '@storybook/blocks';
import * as ButtonStories from './Button.stories';

<Meta of={ButtonStories} />

# Button

Buttons trigger actions. Use the appropriate variant to communicate the
importance of the action.

## When to use

- **Primary**: For the main action on a page (1 per page maximum)
- **Secondary**: For supporting actions
- **Destructive**: For irreversible actions (delete, remove)
- **Ghost**: For low-emphasis actions in toolbars
- **Outline**: For secondary actions that need more visual weight than ghost

## Do's and Don'ts

| Do | Don't |
|---|---|
| Use verb-first labels ("Save changes") | Don't use vague labels ("Click here") |
| Use one primary button per page section | Don't use multiple primary buttons |
| Disable button while form is submitting | Don't remove the button during loading |
| Show loading state for async actions | Don't leave user wondering if click worked |

## Examples

<Canvas of={ButtonStories.Primary} />
<Controls of={ButtonStories.Primary} />

## Accessibility

- All buttons must have visible text or `aria-label`
- Loading state must be announced to screen readers
- Disabled buttons remain in tab order with `aria-disabled`
- Color contrast meets WCAG AA (4.5:1 for text)
```

## 7. Interaction Testing (Play Functions)

```tsx
// Button.stories.tsx — play function for interaction testing
import { within, userEvent, expect } from '@storybook/test';

export const FormSubmission: Story = {
  render: () => (
    <form onSubmit={(e) => { e.preventDefault(); }}>
      <label htmlFor="name">Name</label>
      <input id="name" required />
      <Button type="submit">Submit</Button>
    </form>
  ),
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement);

    // Step 1: Try submitting empty form
    await userEvent.click(canvas.getByRole('button', { name: 'Submit' }));

    // Step 2: Fill in the form
    await userEvent.type(canvas.getByLabelText('Name'), 'Jane Doe');

    // Step 3: Submit
    await userEvent.click(canvas.getByRole('button', { name: 'Submit' }));

    // Step 4: Verify button text
    await expect(canvas.getByRole('button')).toHaveTextContent('Submit');
  },
};

export const KeyboardNavigation: Story = {
  render: () => (
    <div>
      <Button variant="primary">First</Button>
      <Button variant="secondary">Second</Button>
      <Button variant="ghost">Third</Button>
    </div>
  ),
  play: async ({ canvasElement }) => {
    const canvas = within(canvasElement);

    // Tab through buttons
    await userEvent.tab();
    await expect(canvas.getByText('First')).toHaveFocus();

    await userEvent.tab();
    await expect(canvas.getByText('Second')).toHaveFocus();

    await userEvent.tab();
    await expect(canvas.getByText('Third')).toHaveFocus();

    // Activate with keyboard
    await userEvent.keyboard('{Enter}');
  },
};
```

## 9. Anti-Patterns

| Anti-Pattern | Symptom | Fix |
|---|---|---|
| **No documentation** | Developers don't know components exist, build custom | Storybook with autodocs for EVERY component |
| **Docs-only, no live examples** | Developers can't see component in action | Every doc page has Canvas with live, interactive example |
| **No usage guidelines** | Components used incorrectly (primary button everywhere) | MDX guidelines: when to use, do's/don'ts, accessibility |
| **Stale documentation** | Docs don't match current API — code changed, docs didn't | Auto-generate props from TypeScript + CI check for stories |
| **No changelog** | Teams don't know what changed, afraid to upgrade | SemVer changelog with migration guides for breaking changes |
| **No Figma link** | Designers and developers reference different sources | Link Figma frames in Storybook with addon-designs |
| **No search** | Can't find components in large design system | Storybook's built-in search + organized story hierarchy |
