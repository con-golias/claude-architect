## Introduction

AI coding assistants have transformed software development, enabling engineers to generate
code at unprecedented speed. GitHub Copilot, Amazon CodeWhisperer, Claude Code, and similar
tools now generate a significant portion of production code in many organizations. However,
this velocity comes with a critical security cost: studies consistently demonstrate that
AI-generated code contains vulnerabilities at alarming rates.

This guide provides a comprehensive framework for identifying, preventing, and mitigating
security vulnerabilities in AI-generated code. It establishes processes and tooling that
enable teams to benefit from AI coding assistance without compromising security posture.

The fundamental principle is clear: **treat all AI-generated code as untrusted input that
requires security review before deployment.**

---

## The Scale of AI-Generated Code Vulnerabilities

### Research Findings

Multiple peer-reviewed studies and industry analyses have quantified the security risks
of AI-generated code:

- **Stanford University (2022)**: Participants using AI coding assistants produced
  significantly less secure code, yet were more confident in the security of their output.
- **NYU Study**: Found that approximately 40% of code generated by Copilot contained
  exploitable vulnerabilities.
- **Security Research Meta-Analysis**: Across multiple studies, 40-62% of AI-generated
  code snippets contain at least one security vulnerability.
- **GitClear (2024)**: Reported a sharp increase in "code churn" -- code that is quickly
  revised or reverted -- correlating with AI code generation adoption.

### Categories of Risk

```
Risk Severity Distribution in AI-Generated Code
================================================

Critical (Remote Code Execution, SQLi)     : ~15% of vulnerable outputs
High (Authentication Bypass, XSS)          : ~25% of vulnerable outputs
Medium (Information Disclosure, SSRF)       : ~35% of vulnerable outputs
Low (Missing Headers, Verbose Errors)      : ~25% of vulnerable outputs
```

### The Confidence Problem

One of the most dangerous aspects of AI-generated code is that developers tend to trust it
more than they should. Studies show:

- Developers using AI assistants rate their code as more secure than non-AI-assisted code.
- Developers spend less time reviewing AI-generated code than hand-written code.
- The fluency and formatting of AI-generated code creates a false sense of correctness.
- Copy-paste behavior is amplified when code "looks right."

## Why AI Generates Insecure Code

Understanding the root causes of AI code vulnerabilities is essential for building effective
defenses.

### 1. Trained on Vulnerable Code

Large language models learn from massive datasets of publicly available code, including
GitHub repositories, Stack Overflow answers, and documentation. A significant portion of
this training data contains security vulnerabilities:

```
Training Data Composition (Approximate)
========================================
Public GitHub repositories       : Contains known CVEs, unmaintained code
Stack Overflow answers           : Optimized for brevity, not security
Tutorial/blog code               : Simplified examples lacking production hardening
Legacy codebases                 : Pre-modern-security-practice patterns
Documentation examples           : Functional demonstrations, not security references
```

The model learns to reproduce the most common patterns, and insecure patterns are
disproportionately represented in public code.

### 2. No Context About Security Requirements

AI models generate code based on the immediate prompt context. They lack awareness of:

- The application's threat model
- Compliance requirements (PCI-DSS, HIPAA, SOC 2)
- The deployment environment's security architecture
- Data sensitivity classifications
- Organization-specific security policies
- Authentication and authorization context

### 3. Lack of Threat Awareness

AI models do not reason about adversarial scenarios. They generate code that fulfills
functional requirements but do not consider:

- How an attacker might manipulate inputs
- What happens when assumptions are violated
- Side-channel information leakage
- Race conditions and time-of-check-time-of-use (TOCTOU) vulnerabilities
- Privilege escalation paths

### 4. Pattern Matching Without Understanding

LLMs perform statistical pattern matching, not semantic understanding. This means:

- They reproduce patterns without understanding why a pattern is secure or insecure.
- They cannot evaluate whether a security control is appropriate for the context.
- They optimize for "most likely next token" not "most secure next token."
- They may combine individually safe patterns in unsafe ways.

### 5. Stale Security Knowledge

Models have training data cutoffs and may not reflect:

- Recently discovered vulnerability classes
- Updated library APIs that fix security issues
- Deprecated cryptographic algorithms
- New security best practices and standards

## Common Vulnerability Patterns in AI-Generated Code

### Pattern 1: Hardcoded Secrets

AI models frequently embed secrets directly in generated code because training data
is full of examples with hardcoded credentials.

```python

# AI commonly generates code like this

    's3',
    aws_access_key_id='AKIAIOSFODNN7EXAMPLE',
    aws_secret_access_key='wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY',
    region_name='us-east-1'
)

# Database connection with hardcoded credentials

# API key embedded in source
```


### Pattern 2: SQL Injection

AI models routinely generate string-concatenated SQL queries:

```python
# VULNERABLE: AI-generated SQL injection
def get_user(username):

    cursor.execute(query)
    return cursor.fetchone()

def search_products(category, min_price):

    return cursor.fetchall()
```

```python
# SECURE: Parameterized queries
def get_user(username: str):

    cursor.execute(query, (username,))
    return cursor.fetchone()

def search_products(category: str, min_price: float):
    query = "SELECT * FROM products WHERE category = %s AND price > %s"
    cursor.execute(query, (category, min_price))

# Using an ORM (SQLAlchemy) for additional safety
def get_user_orm(username: str):
    return session.query(User).filter(User.username == username).first()
```

```javascript
// SECURE: Path traversal prevention
const express = require('express');
const path = require('path');
const fs = require('fs');

const UPLOAD_DIR = path.resolve('/uploads');
const LOG_DIR = path.resolve('./logs');

app.get('/download', (req, res) => {
    const filename = req.query.file;

    if (!filename || typeof filename !== 'string') {
        return res.status(400).json({ error: 'Invalid filename' });
    }

    // Remove path separators and null bytes
    const sanitized = path.basename(filename).replace(/\0/g, '');

    // Resolve and verify the path stays within the allowed directory
    const filepath = path.resolve(UPLOAD_DIR, sanitized);

    if (!filepath.startsWith(UPLOAD_DIR)) {
        return res.status(403).json({ error: 'Access denied' });
    }

    // Verify file exists before sending
    if (!fs.existsSync(filepath)) {
        return res.status(404).json({ error: 'File not found' });
    }

    res.sendFile(filepath);
});

app.get('/api/logs', (req, res) => {
    const logFile = req.query.name;

    if (!logFile || typeof logFile !== 'string') {
        return res.status(400).json({ error: 'Invalid log file name' });
    }

    // Allow only alphanumeric names with .log extension
    if (!/^[a-zA-Z0-9_-]+\.log$/.test(logFile)) {
        return res.status(400).json({ error: 'Invalid log file format' });
    }

    const filepath = path.resolve(LOG_DIR, logFile);

    if (!filepath.startsWith(LOG_DIR)) {
        return res.status(403).json({ error: 'Access denied' });
    }

    try {
        const content = fs.readFileSync(filepath, 'utf8');
        res.type('text/plain').send(content);
    } catch (err) {
        res.status(404).json({ error: 'Log file not found' });
    }
});
```

### Pattern 4: Insecure Deserialization

AI commonly generates unsafe deserialization patterns:

```python
# VULNERABLE: AI-generated insecure deserialization
import pickle
import yaml

def load_config(config_path):
    with open(config_path, 'rb') as f:
        return pickle.load(f)  # Arbitrary code execution

def parse_yaml_input(user_input):
    return yaml.load(user_input)  # yaml.load without Loader is unsafe

def load_user_session(session_data):
    import marshal
    return marshal.loads(session_data)  # Unsafe deserialization
```

```python
# SECURE: Safe deserialization practices
import json
import yaml
from typing import Any, Dict

def load_config(config_path: str) -> Dict[str, Any]:
    """Load configuration from a JSON file (no code execution risk)."""
    with open(config_path, 'r') as f:
        config = json.load(f)
    validate_config_schema(config)
    return config

def parse_yaml_input(user_input: str) -> Dict[str, Any]:
    """Parse YAML safely using SafeLoader."""
    return yaml.safe_load(user_input)

def load_user_session(session_data: str) -> Dict[str, Any]:
    """Deserialize session data using a safe format with validation."""
    try:
        data = json.loads(session_data)
    except json.JSONDecodeError:
        raise ValueError("Invalid session data format")

    # Validate expected schema
    required_keys = {'user_id', 'created_at', 'permissions'}
    if not required_keys.issubset(data.keys()):
        raise ValueError("Session data missing required fields")

    return data
```

### Pattern 5: Weak Cryptography

AI models frequently suggest outdated or weak cryptographic practices:

```python
# VULNERABLE: AI-generated weak cryptography
import hashlib
import random
import string

def hash_password(password):
    return hashlib.md5(password.encode()).hexdigest()

def generate_token():
    chars = string.ascii_letters + string.digits
    return ''.join(random.choice(chars) for _ in range(16))

def encrypt_data(data, key):
    from Crypto.Cipher import DES
    cipher = DES.new(key.encode()[:8], DES.MODE_ECB)
    padded = data + ' ' * (8 - len(data) % 8)
    return cipher.encrypt(padded.encode())
```

```python
# SECURE: Modern cryptographic practices
import bcrypt
import secrets
from cryptography.fernet import Fernet
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os

def hash_password(password: str) -> str:
    """Hash password using bcrypt with automatic salt generation."""
    salt = bcrypt.gensalt(rounds=12)
    return bcrypt.hashpw(password.encode('utf-8'), salt).decode('utf-8')

def verify_password(password: str, hashed: str) -> bool:
    """Verify a password against its bcrypt hash."""
    return bcrypt.checkpw(password.encode('utf-8'), hashed.encode('utf-8'))

def generate_token(length: int = 32) -> str:
    """Generate a cryptographically secure random token."""
    return secrets.token_urlsafe(length)

def encrypt_data(data: bytes, key: bytes) -> bytes:
    """Encrypt data using AES-256-GCM with authenticated encryption."""
    nonce = os.urandom(12)
    aesgcm = AESGCM(key)  # key must be 32 bytes for AES-256
    ciphertext = aesgcm.encrypt(nonce, data, None)
    return nonce + ciphertext  # Prepend nonce for decryption

def decrypt_data(encrypted: bytes, key: bytes) -> bytes:
    """Decrypt AES-256-GCM encrypted data."""
    nonce = encrypted[:12]
    ciphertext = encrypted[12:]
    aesgcm = AESGCM(key)
    return aesgcm.decrypt(nonce, ciphertext, None)
```

```python
# SECURE: Comprehensive input validation
from flask import Flask, request, jsonify
from marshmallow import Schema, fields, validate, ValidationError
from bleach import clean
import re

app = Flask(__name__)

class CreateUserSchema(Schema):
    name = fields.String(
        required=True,
        validate=[
            validate.Length(min=1, max=100),
            validate.Regexp(r'^[a-zA-Z\s\'-]+$', error='Name contains invalid characters')
        ]
    )
    email = fields.Email(required=True, validate=validate.Length(max=254))
    age = fields.Integer(
        required=True,
        validate=validate.Range(min=13, max=150)
    )
    role = fields.String(
        required=True,
        validate=validate.OneOf(['user', 'editor', 'viewer'])
    )

create_user_schema = CreateUserSchema()

ALLOWED_ROLES = {'user', 'editor', 'viewer'}  # 'admin' explicitly excluded

@app.route('/api/users', methods=['POST'])
def create_user():
    # Validate content type
    if not request.is_json:
        return jsonify({'error': 'Content-Type must be application/json'}), 415

    # Parse and validate input
    try:
        data = create_user_schema.load(request.json)
    except ValidationError as err:
        return jsonify({'errors': err.messages}), 400

    # Additional server-side authorization check
    if data['role'] not in ALLOWED_ROLES:
        return jsonify({'error': 'Invalid role assignment'}), 403

    # Sanitize string fields
    sanitized_name = clean(data['name'], tags=[], strip=True)

    user = User(
        name=sanitized_name,
        email=data['email'].lower().strip(),
        age=data['age'],
        role=data['role']
    )
    db.session.add(user)
    db.session.commit()

    return jsonify({'id': user.id}), 201
```

### Pattern 7: Cross-Site Scripting (XSS)

AI-generated frontend code often fails to sanitize output:

```javascript
// VULNERABLE: AI-generated XSS
function displayUserProfile(user) {
    document.getElementById('profile').innerHTML = `
        <h1>${user.name}</h1>
        <p>${user.bio}</p>
        <div>${user.website}</div>
    `;
}

// AI-generated React component with dangerouslySetInnerHTML
function Comment({ comment }) {
    return (
        <div dangerouslySetInnerHTML={{ __html: comment.body }} />
    );
}
```

```javascript
// SECURE: Proper output encoding and sanitization
import DOMPurify from 'dompurify';

function displayUserProfile(user) {
    const profileEl = document.getElementById('profile');

    // Use textContent for plain text (automatically encodes)
    const h1 = document.createElement('h1');
    h1.textContent = user.name;

    const bio = document.createElement('p');
    bio.textContent = user.bio;

    // Validate and sanitize URLs
    const websiteDiv = document.createElement('div');
    if (isValidHttpUrl(user.website)) {
        const link = document.createElement('a');
        link.href = user.website;
        link.textContent = user.website;
        link.rel = 'noopener noreferrer';
        link.target = '_blank';
        websiteDiv.appendChild(link);
    }

    profileEl.replaceChildren(h1, bio, websiteDiv);
}

function isValidHttpUrl(string) {
    try {
        const url = new URL(string);
        return url.protocol === 'http:' || url.protocol === 'https:';
    } catch {
        return false;
    }
}

// Safe React component using DOMPurify when HTML rendering is required
function Comment({ comment }) {
    const sanitizedHtml = DOMPurify.sanitize(comment.body, {
        ALLOWED_TAGS: ['b', 'i', 'em', 'strong', 'p', 'br', 'ul', 'ol', 'li'],
        ALLOWED_ATTR: []
    });

    return (
        <div dangerouslySetInnerHTML={{ __html: sanitizedHtml }} />
    );
}

// Preferred: Use markdown rendering instead of raw HTML
import ReactMarkdown from 'react-markdown';

function SafeComment({ comment }) {
    return (
        <div className="comment">
            <ReactMarkdown>{comment.body}</ReactMarkdown>
        </div>
    );
}
```

## Secure AI-Assisted Coding Practices

### Always Review Generated Code for Security

Establish a mandatory security review process for all AI-generated code:

```
AI Code Security Review Protocol
=================================
1. Identify AI-generated code blocks (use comments or tracking)
2. Check for hardcoded secrets and credentials
3. Verify input validation on all external inputs
4. Confirm parameterized queries for all database operations
5. Validate output encoding for all rendered content
6. Check cryptographic function choices
7. Review error handling (no stack traces to users)
8. Verify authentication and authorization checks
9. Check file and path operations for traversal
10. Run SAST tools on the generated code
```

### Use Security-Focused Prompts

Frame prompts to include security requirements explicitly:

```
POOR PROMPT:
"Write a function to authenticate users"

BETTER PROMPT:
"Write a user authentication function with the following security requirements:
- Use bcrypt for password hashing with a work factor of 12
- Implement rate limiting (max 5 attempts per 15 minutes per IP)
- Use parameterized queries for database lookups
- Return generic error messages (do not reveal if username exists)
- Log failed authentication attempts with IP and timestamp
- Set secure session cookie attributes (HttpOnly, Secure, SameSite)
- Implement account lockout after 10 consecutive failures"

POOR PROMPT:
"Create an API endpoint to search products"

BETTER PROMPT:
"Create a REST API endpoint to search products with these security controls:
- Validate and sanitize all query parameters
- Use parameterized SQL queries (never string concatenation)
- Limit results to 100 per page to prevent DoS
- Only return fields the user is authorized to see
- Set appropriate response headers (Content-Type, X-Content-Type-Options)
- Handle errors without exposing internal details
- Rate limit to 60 requests per minute per API key"
```

## Security-Focused Prompt Engineering

### Prompt Templates for Secure Code Generation

```markdown
## Template: Secure API Endpoint

Generate a {language} API endpoint for {functionality} with these security controls:

**Input Validation:**
- Validate all parameters against a strict schema
- Reject unexpected fields
- Enforce type, length, and format constraints

**Authentication/Authorization:**
- Require authentication via {auth_method}
- Check authorization for the specific resource
- Return 403 for unauthorized access, 401 for unauthenticated

**Data Access:**
- Use parameterized queries only
- Apply principle of least privilege for database queries
- Return only necessary fields

**Error Handling:**
- Return generic error messages to the client
- Log detailed errors server-side
- Never expose stack traces, database schemas, or internal paths

**Output:**
- Set Content-Type header explicitly
- Encode output appropriately for the context
- Include security headers
```

```markdown
## Template: Secure File Operations

Generate a {language} function for {file_operation} with these security controls:

**Path Validation:**
- Resolve paths to absolute form and verify within allowed directory
- Reject path traversal sequences (../, ..\, null bytes)
- Use allowlist for permitted file extensions

**File Content:**
- Validate file size before processing
- Check file type by content (magic bytes), not just extension
- Scan for malicious content if processing uploads

**Permissions:**
- Use least-privilege file system permissions
- Do not create world-readable/writable files
- Validate the requesting user has access to the file
```

## AI Coding Assistants Security Features

### GitHub Copilot

GitHub Copilot includes several security-relevant features:

- **Code Referencing**: Identifies when generated code closely matches public code,
  helping detect license and potential security issues from known-vulnerable code.
- **Vulnerability Filtering**: Filters out some common vulnerability patterns from
  suggestions (hardcoded credentials, SQL injection in simple cases).
- **Copilot Chat Security Analysis**: Can analyze code for security issues when prompted.

Limitations:
- Vulnerability filtering is not comprehensive.
- Does not understand the application's threat model.
- May suggest deprecated or vulnerable library versions.

### Amazon CodeWhisperer

CodeWhisperer provides:

- **Built-in Security Scanning**: Scans generated code for vulnerabilities mapped to
  CWE and OWASP categories.
- **Secrets Detection**: Flags hardcoded credentials in generated code.
- **Reference Tracking**: Tracks code that matches open-source references.

### Claude Code

Claude Code offers:

- **Security Awareness**: Trained to consider security implications and can flag potential
  issues in generated code when prompted.
- **Contextual Understanding**: Can incorporate security requirements from project context.
- **Permission Model**: Requires explicit approval for file system and command operations.

### Automated AI Code Review Pipeline

```python
# scripts/ai_code_review.py
"""
Automated security review pipeline for AI-generated code.
Integrates SAST tools and produces a unified security report.
"""
import subprocess
import json
import sys
from dataclasses import dataclass
from typing import List
from pathlib import Path


@dataclass
class SecurityFinding:
    tool: str
    severity: str
    file: str
    line: int
    rule_id: str
    message: str
    cwe: str = ""


def run_semgrep(target_path: str) -> List[SecurityFinding]:
    """Run Semgrep security scan."""
    result = subprocess.run(
        [
            'semgrep', 'scan',
            '--config', 'p/owasp-top-ten',
            '--config', 'p/secrets',
            '--json',
            target_path,
        ],
        capture_output=True, text=True
    )

    findings = []
    if result.stdout:
        data = json.loads(result.stdout)
        for match in data.get('results', []):
            findings.append(SecurityFinding(
                tool='semgrep',
                severity=match.get('extra', {}).get('severity', 'UNKNOWN'),
                file=match.get('path', ''),
                line=match.get('start', {}).get('line', 0),
                rule_id=match.get('check_id', ''),
                message=match.get('extra', {}).get('message', ''),
                cwe=str(match.get('extra', {}).get('metadata', {}).get('cwe', '')),
            ))
    return findings


def run_bandit(target_path: str) -> List[SecurityFinding]:
    """Run Bandit security scan for Python code."""
    result = subprocess.run(
        ['bandit', '-r', target_path, '-f', 'json'],
        capture_output=True, text=True
    )

    findings = []
    if result.stdout:
        data = json.loads(result.stdout)
        for issue in data.get('results', []):
            findings.append(SecurityFinding(
                tool='bandit',
                severity=issue.get('issue_severity', 'UNKNOWN'),
                file=issue.get('filename', ''),
                line=issue.get('line_number', 0),
                rule_id=issue.get('test_id', ''),
                message=issue.get('issue_text', ''),
                cwe=str(issue.get('issue_cwe', {}).get('id', '')),
            ))
    return findings


def generate_report(findings: List[SecurityFinding]) -> dict:
    """Generate a unified security report."""
    critical = [f for f in findings if f.severity in ('ERROR', 'HIGH', 'CRITICAL')]
    return {
        'total_findings': len(findings),
        'critical_findings': len(critical),
        'pass': len(critical) == 0,
        'findings_by_severity': {
            'critical': len([f for f in findings if f.severity == 'CRITICAL']),
            'high': len([f for f in findings if f.severity in ('HIGH', 'ERROR')]),
            'medium': len([f for f in findings if f.severity in ('MEDIUM', 'WARNING')]),
            'low': len([f for f in findings if f.severity == 'LOW']),
        },
        'findings': [
            {
                'tool': f.tool,
                'severity': f.severity,
                'file': f.file,
                'line': f.line,
                'rule': f.rule_id,
                'message': f.message,
                'cwe': f.cwe,
            }
            for f in findings
        ],
    }


def main():
    target = sys.argv[1] if len(sys.argv) > 1 else '.'

    all_findings = []
    all_findings.extend(run_semgrep(target))
    all_findings.extend(run_bandit(target))

    report = generate_report(all_findings)
    print(json.dumps(report, indent=2))

    if not report['pass']:
        print(f"\nSECURITY GATE FAILED: {report['critical_findings']} critical findings")
        sys.exit(1)
    else:
        print("\nSecurity gate passed.")
        sys.exit(0)


if __name__ == '__main__':
    main()
```

```python
# SECURE: Corrected authentication endpoint

import os
from datetime import datetime, timedelta, timezone
from functools import wraps

import bcrypt
import jwt
from flask import Flask, request, jsonify, g
from flask_limiter import Limiter
from flask_limiter.util import get_remote_address
from flask_cors import CORS
from flask_talisman import Talisman
from marshmallow import Schema, fields, validate, ValidationError
import psycopg2
from psycopg2.extras import RealDictCursor

app = Flask(__name__)
app.config['SECRET_KEY'] = os.environ['FLASK_SECRET_KEY']
app.config['JWT_SECRET'] = os.environ['JWT_SECRET_KEY']
app.config['JWT_EXPIRATION_MINUTES'] = 15
app.config['JWT_REFRESH_EXPIRATION_DAYS'] = 7

# Security middleware
Talisman(app, force_https=True)
CORS(app, origins=os.environ.get('ALLOWED_ORIGINS', '').split(','))
limiter = Limiter(app=app, key_func=get_remote_address, default_limits=["100/hour"])

# Input validation schemas
class LoginSchema(Schema):
    username = fields.String(
        required=True,
        validate=[
            validate.Length(min=3, max=50),
            validate.Regexp(r'^[a-zA-Z0-9_]+$')
        ]
    )
    password = fields.String(
        required=True,
        validate=validate.Length(min=8, max=128)
    )

login_schema = LoginSchema()


def get_db():
    if 'db' not in g:
        g.db = psycopg2.connect(
            os.environ['DATABASE_URL'],
            cursor_factory=RealDictCursor
        )
    return g.db


@app.teardown_appcontext
def close_db(exception):
    db = g.pop('db', None)
    if db is not None:
        db.close()


def require_auth(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        auth_header = request.headers.get('Authorization', '')
        if not auth_header.startswith('Bearer '):
            return jsonify({'error': 'Authentication required'}), 401

        token = auth_header[7:]  # Remove 'Bearer ' prefix
        try:
            payload = jwt.decode(
                token,
                app.config['JWT_SECRET'],
                algorithms=['HS256'],
                options={
                    'require': ['exp', 'iat', 'sub', 'iss'],
                    'verify_exp': True,
                    'verify_iss': True,
                },
                issuer='myapp.example.com',
            )
            g.current_user_id = payload['sub']
        except jwt.ExpiredSignatureError:
            return jsonify({'error': 'Token expired'}), 401
        except jwt.InvalidTokenError:
            return jsonify({'error': 'Invalid token'}), 401

        return f(*args, **kwargs)
    return decorated


@app.route('/api/v1/auth/login', methods=['POST'])
@limiter.limit("5/minute")
def login():
    # Validate content type
    if not request.is_json:
        return jsonify({'error': 'Content-Type must be application/json'}), 415

    # Validate input
    try:
        data = login_schema.load(request.json)
    except ValidationError as err:
        return jsonify({'error': 'Invalid input', 'details': err.messages}), 400

    # Fetch user with parameterized query
    db = get_db()
    cursor = db.cursor()
    cursor.execute(
        "SELECT id, username, password_hash FROM users WHERE username = %s AND active = true",
        (data['username'],)
    )
    user = cursor.fetchone()

    # Generic error message prevents username enumeration
    generic_error = {'error': 'Invalid username or password'}

    if user is None:
        # Perform dummy bcrypt check to prevent timing attacks
        bcrypt.checkpw(b'dummy', bcrypt.hashpw(b'dummy', bcrypt.gensalt()))
        return jsonify(generic_error), 401

    # Verify password with bcrypt
    if not bcrypt.checkpw(
        data['password'].encode('utf-8'),
        user['password_hash'].encode('utf-8')
    ):
        # Log failed attempt
        app.logger.warning(
            'Failed login attempt for user %s from IP %s',
            data['username'],
            request.remote_addr,
        )

        # Increment failed attempts (lockout logic)
        cursor.execute(
            "UPDATE users SET failed_login_attempts = failed_login_attempts + 1 "
            "WHERE id = %s",
            (user['id'],)
        )
        db.commit()

        return jsonify(generic_error), 401

    # Reset failed attempts on successful login
    cursor.execute(
        "UPDATE users SET failed_login_attempts = 0, last_login = NOW() WHERE id = %s",
        (user['id'],)
    )
    db.commit()

    # Generate JWT with proper claims
    now = datetime.now(timezone.utc)
    access_token = jwt.encode(
        {
            'sub': user['id'],
            'iss': 'myapp.example.com',
            'iat': now,
            'exp': now + timedelta(minutes=app.config['JWT_EXPIRATION_MINUTES']),
        },
        app.config['JWT_SECRET'],
        algorithm='HS256',
    )

    app.logger.info('Successful login for user %s', data['username'])

    return jsonify({
        'access_token': access_token,
        'token_type': 'Bearer',
        'expires_in': app.config['JWT_EXPIRATION_MINUTES'] * 60,
    })


@app.route('/api/v1/users/me', methods=['GET'])
@require_auth
def profile():
    db = get_db()
    cursor = db.cursor()
    # Parameterized query - only return safe fields
    cursor.execute(
        "SELECT id, username, email, created_at FROM users WHERE id = %s",
        (g.current_user_id,)
    )
    user = cursor.fetchone()

    if user is None:
        return jsonify({'error': 'User not found'}), 404

    # Return only non-sensitive fields (no password hash)
    return jsonify({
        'id': user['id'],
        'username': user['username'],
        'email': user['email'],
        'created_at': user['created_at'].isoformat(),
    })
```

## Best Practices

### 1. Treat AI-Generated Code as Untrusted Input
Never deploy AI-generated code without human security review. Establish the same rigor
applied to third-party libraries or user-submitted code.

### 2. Embed Security Requirements in Every Prompt
Always include explicit security requirements in AI coding prompts: input validation,
parameterized queries, output encoding, authentication checks, and error handling.

### 3. Run SAST Tools on All AI-Generated Code
Configure Semgrep, CodeQL, Bandit, ESLint security plugins, and secrets scanners in CI/CD
pipelines. Make security gates blocking for critical and high findings.

### 4. Never Trust AI with Security-Critical Logic Without Expert Review
Authentication, authorization, cryptography, and access control code generated by AI must
be reviewed by security-knowledgeable engineers before deployment.

### 5. Use AI to Find Vulnerabilities, Not Just Generate Code
Leverage AI assistants for security auditing: ask them to review code for vulnerabilities,
suggest security improvements, and identify missing security controls.

### 6. Maintain a Security Prompt Library
Create and share security-focused prompt templates across the team. Standardize how
security requirements are communicated to AI assistants.

### 7. Track AI-Generated Code
Label or annotate AI-generated code sections so reviewers know to apply extra security
scrutiny. Use code comments, commit messages, or metadata tracking.

### 8. Layer Multiple Defenses
Do not rely on any single control. Combine secure prompts, human review, SAST, DAST,
secrets scanning, and runtime protection for defense in depth.

### 9. Keep SAST Rules Updated for AI Patterns
Update static analysis rules regularly to detect new patterns of AI-generated
vulnerabilities as they are discovered.

### 10. Educate Developers on AI Security Risks
Train all developers who use AI coding assistants on the specific vulnerability patterns
these tools produce. Make AI code security part of onboarding and ongoing training.

---

## Anti-Patterns

### 1. Blindly Trusting AI-Generated Code
Accepting AI suggestions without review because "the AI knows best." AI models optimize
for plausibility, not security. Every suggestion must be evaluated.

### 2. Security by Assumption
Assuming the AI coding assistant has already checked for security issues. Most AI assistants
do not perform comprehensive security analysis of their own output.

### 3. Copy-Paste Without Context Adaptation
Copying AI-generated code snippets into production without adapting them to the application's
specific security requirements, threat model, and architecture.

### 4. Using AI for Cryptographic Implementation
Allowing AI to implement custom cryptographic protocols, key derivation, or encryption
schemes. Always use established libraries and have implementations reviewed by experts.

### 5. Disabling Security Tools to Reduce Noise
Turning off SAST rules or pre-commit hooks because AI-generated code triggers too many
findings. The findings exist because the code is insecure.

### 6. AI-Generated Security Configurations
Deploying AI-generated security configurations (CORS, CSP, authentication) without
understanding and validating every setting.

### 7. Using AI-Generated Regex for Security Validation
Relying on AI-generated regular expressions for security-critical input validation. AI regex
is frequently incorrect, incomplete, or vulnerable to ReDoS.

### 8. Prompting Without Security Context
Asking AI to generate code without specifying security requirements, then assuming the
output is secure. The model will optimize for functionality, not security.
